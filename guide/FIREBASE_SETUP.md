# 🔥 Firebase Setup Guide

Complete Firebase configuration needed for the AI Exam Monitor project.

2. Click "Create project"
3. Project name: `ai-exam-monitor`
4. Click "Continue"
5. Select/Create Google Cloud project
6. Enable Google Analytics (optional)
7. Click "Create project"
8. Wait for project creation (2-3 minutes)

## Step 2: Get Firebase Service Account Credentials

### For Backend (Render):

1. Go to Firebase Project Dashboard
2. Click Settings (gear icon) ⚙️
3. Go to "Service Accounts" tab
4. Language: Select "Node.js"
5. Click "Generate new private key"
6. A JSON file downloads - this is your service account

### JSON Structure:

```json
{
  "type": "service_account",
  "project_id": "your-project-id",
  "private_key_id": "...",
  "private_key": "-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n",
  "client_email": "...",
  "client_id": "...",
  "auth_uri": "https://accounts.google.com/o/oauth2/auth",
  "token_uri": "https://oauth2.googleapis.com/token",
  "auth_provider_x509_cert_url": "https://www.googleapis.com/oauth2/v1/certs",
  "client_x509_cert_url": "...",
  "universe_domain": "googleapis.com"
}
```

### Extract Values:

Open the downloaded JSON and copy these values (keep them secure!):

- `type`
- `project_id`
- `private_key_id`
- `private_key` (the FULL key including BEGIN/END)
- `client_email`
- `client_id`
- `auth_uri`
- `token_uri`
- `auth_provider_x509_cert_url`
- `client_x509_cert_url`
- `universe_domain`

These go into backend environment variables on Render.

3. Go to "General" tab

## Step 4: Set Up Firestore Database

### Create Firestore Database:

1. Firebase Dashboard → Your Project
2. Left sidebar → "Firestore Database"
3. Click "Create database"
4. Location: Choose closest region (e.g., `us-central1`)
5. Security rules: Start in **"Test mode"** for development
6. Click "Create"

### Test Mode Rules (Development Only):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

This allows any reads/writes. Change this for production!

## Step 6: Configure Firestore Collections

These collections will be created automatically when users sign up:

### 1. `users` Collection

```javascript
// Document ID: user ID (auto-generated by Firebase Auth)
{
  email: "student@example.com",
  role: "student", // or "faculty" or "admin"
  name: "John Doe",
  phone: "1234567890",
  createdAt: Timestamp,
  lastLogin: Timestamp,
  isActive: true
}
```

### 2. `exams` Collection

```javascript
{
  title: "Final Exam - Math",
  description: "Calculus Final Exam",
  subject: "Mathematics",
  duration: 120, // minutes
  totalQuestions: 50,
  totalMarks: 100,
  startTime: Timestamp,
  endTime: Timestamp,
  createdBy: "faculty_user_id",
  isPublished: false,
  passingMarks: 40,
  createdAt: Timestamp,
  negative_marks_per_wrong: 0.25
}
```

### 3. `questions` Collection

```javascript
{
  examId: "exam_id",
  type: "multiple_choice", // or "short_answer", "essay"
  question: "What is 2+2?",
  options: ["3", "4", "5", "6"], // for multiple choice
  correctAnswer: "4", // or array for multiple select
  marks: 2,
  difficulty: "easy", // easy, medium, hard
  explanation: "2+2 equals 4 because...",
  order: 1
}
```

### 4. `submissions` Collection

```javascript
{
  examId: "exam_id",
  studentId: "student_user_id",
  answers: {
    "question_id_1": "selected_option",
    "question_id_2": "their_answer"
  },
  marksObtained: 85,
  totalMarks: 100,
  status: "submitted", // draft, submitted, evaluated
  startTime: Timestamp,
  endTime: Timestamp,
  duration: 120, // minutes
  deviceInfo: {
    fingerprint: "unique_device_id",
    browser: "Chrome",
    os: "Windows 10"
  },
  proctorSessions: [
    {
      type: "video_monitoring",
      startTime: Timestamp,
      endTime: Timestamp,
      status: "active"
    }
  ]
}
```

### 5. `proctorLogs` Collection

```javascript
{
  examId: "exam_id",
  studentId: "student_user_id",
  timestamp: Timestamp,
  event: "tab_switch", // or "screen_leave", "wrong_device", etc.
  details: "Switched to another tab",
  severity: "warning" // info, warning, critical
}
```

## Step 9: Configure Security Rules (IMPORTANT!)

### Development Rules (Test Mode):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

⚠️ This allows anyone to read/write everything. For development only!

### Production Rules (Secure):

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    // Public data
    match /exams/{examId} {
      allow read: if request.auth != null;
      allow write: if request.auth.token.role == 'faculty' || request.auth.token.role == 'admin';
    }

    // User profiles
    match /users/{userId} {
      allow read: if request.auth.uid == userId;
      allow write: if request.auth.uid == userId;
    }

    // Student submissions (only see own)
    match /submissions/{submissionId} {
      allow read: if request.auth.uid == resource.data.studentId
                     || request.auth.token.role == 'faculty';
      allow write: if request.auth.uid == resource.data.studentId;
    }

    // Questions (only faculty can read)
    match /questions/{questionId} {
      allow read: if request.auth.token.role == 'faculty';
      allow write: if request.auth.token.role == 'faculty';
    }

    // Proctor logs
    match /proctorLogs/{logId} {
      allow read: if request.auth.token.role == 'faculty';
      allow write: if request.auth.token.role == 'faculty' || request.auth.uid == resource.data.studentId;
    }
  }
}
```

## Troubleshooting Firebase Setup

### "Firebase initialization error: Invalid service account"

- Check FIREBASE_PRIVATE_KEY is copied completely
- Ensure all FIREBASE\_\* env vars are present
- Re-download credentials and re-enter them

### "Permission denied" when accessing Firestore

- Check Firestore security rules are not restrictive
- Start with test mode rules (allow all)
- Can't restrict until DB has data structure

### "Collection not found"

- Collections are created automatically when first document is added
- Just use them in code, they'll be created

### "Cannot read Firebase config"

- Frontend env variables need VITE\_ prefix
- Check frontend/.env has all VITE*FIREBASE*\* variables
- Hard refresh browser

## Important Security Notes

1. **Never commit** `.env` or service account JSON
2. **Never share** private keys or credentials
3. **Use HTTPS** in production (Render & Netlify do this)
4. **Update security rules** before going to production
5. **Enable Read-Only Copies** in production deployments
6. **Set up billing alerts** if you go beyond free tier
